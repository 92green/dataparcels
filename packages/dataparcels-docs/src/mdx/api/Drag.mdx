# Drag

```js
import Drag from 'react-dataparcels/Drag';
```

The Drag component provides an easy way to add drag and drop sorting to arrays of items. It uses [react-sortable-hoc](https://github.com/clauderic/react-sortable-hoc) to perform the dragging and sorting, and all of `react-sortable-hoc`s API is accessible to use via Drag.

## <Text textStyle="weaker">Children</Text>

### childRenderer

```flow
(parcel: Parcel) => Node
```
Drag must be given a `childRenderer` function as children. This is called when rendering each child element, passing the Parcel for each element.

* <Param name="parcel" type="Parcel" />

  The Parcel for each element being rendered

## <Text textStyle="weaker">Props</Text>

### source

```flow
source: Parcel
```

The parcel that will be sorted. The parcel's value must be an array.

### container

```flow
container: ComponentType<any> // optional, defaults to 'div'
```

The container element to render around the children. Defaults to `'div'`.

<a id="sortableElementProps" />

### ...sortableElementProps

```flow
...sortableElementProps: any
```
Any additional props will be passed onto [react-sortable-hoc's SortableContainer hoc](https://github.com/clauderic/react-sortable-hoc#sortableelement-hoc), to allow configuration of how the dragging behaves.

Note that `SortableContainer`'s `onSortEnd` prop is not required, as it's provided by Drag. If you pass an `onSortEnd` prop to Drag, it will fire immediately after the Parcel triggers its change request.
